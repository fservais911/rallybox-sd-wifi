# AWS Lambda GPX Signer

## Purpose

This Lambda function creates one fresh presigned S3 `PUT` URL for each GPX filename generated by RallyBox.

Use it when RallyBox uploads to:

```text
tracks/{rallybox_id}/{filename}
```

with a different filename on every export.

For your current bucket, the target object path looks like:

```text
s3://amzn-s3-rallybox-bucket-911/tracks/{rallybox_id}/{filename}
```

## What RallyBox Calls

Recommended signer URL in `sdkconfig`:

```text
CONFIG_RALLYBOX_GPX_WEB_SIGNER_URL="https://YOUR_LAMBDA_URL/gpx-sign?rallybox_id=YOUR_RALLYBOX_ID&filename={filename}"
CONFIG_RALLYBOX_GPX_WEB_SIGNER_AUTH_HEADER="x-api-key"
CONFIG_RALLYBOX_GPX_WEB_SIGNER_AUTH_VALUE="YOUR_SIGNER_SECRET"
```

Example device request:

```text
GET /gpx-sign?rallybox_id=rallyboxid&filename=21042026-124327.gnss.gpx
x-api-key: YOUR_SIGNER_SECRET
```

The Lambda returns JSON like:

```json
{
  "upload_url": "https://amzn-s3-rallybox-bucket-911.s3.eu-central-2.amazonaws.com/tracks/rallyboxid/21042026-124327.gnss.gpx?X-Amz-Algorithm=AWS4-HMAC-SHA256&...",
  "object_key": "tracks/rallyboxid/21042026-124327.gnss.gpx",
  "expires_in": 300
}
```

RallyBox already accepts JSON containing `upload_url`.

## Lambda Python Code

Runtime: Python 3.12 or similar.

```python
import json
import os
import re

import boto3
from botocore.config import Config


S3_BUCKET = os.environ["S3_BUCKET"]
S3_REGION = os.environ.get("S3_REGION", "eu-central-2")
S3_PREFIX = os.environ.get("S3_PREFIX", "tracks")
SIGNER_API_KEY = os.environ["SIGNER_API_KEY"]
URL_EXPIRES_SECONDS = int(os.environ.get("URL_EXPIRES_SECONDS", "300"))

FILENAME_RE = re.compile(r"^[A-Za-z0-9._-]+\.gpx$")
RALLYBOX_ID_RE = re.compile(r"^[A-Za-z0-9._-]+$")

s3 = boto3.client(
  "s3",
  region_name=S3_REGION,
  endpoint_url=f"https://s3.{S3_REGION}.amazonaws.com",
  config=Config(
    signature_version="s3v4",
    s3={"addressing_style": "virtual"},
  ),
)


def response(status_code: int, body: dict) -> dict:
    return {
        "statusCode": status_code,
        "headers": {
            "Content-Type": "application/json"
        },
        "body": json.dumps(body),
    }


def lambda_handler(event, context):
    headers = event.get("headers") or {}
    provided_api_key = headers.get("x-api-key") or headers.get("X-Api-Key") or headers.get("X-API-Key")
    if provided_api_key != SIGNER_API_KEY:
        return response(401, {"error": "unauthorized"})

    params = event.get("queryStringParameters") or {}
    filename = (params.get("filename") or "").strip()
    rallybox_id = (params.get("rallybox_id") or "").strip()

    if not filename or not FILENAME_RE.fullmatch(filename):
        return response(400, {"error": "invalid filename"})

    if not rallybox_id or not RALLYBOX_ID_RE.fullmatch(rallybox_id):
        return response(400, {"error": "invalid rallybox_id"})

    object_key = f"{S3_PREFIX.rstrip('/')}/{rallybox_id}/{filename}"

    upload_url = s3.generate_presigned_url(
        ClientMethod="put_object",
        Params={
            "Bucket": S3_BUCKET,
            "Key": object_key,
            "ContentType": "application/gpx+xml",
        },
        ExpiresIn=URL_EXPIRES_SECONDS,
        HttpMethod="PUT",
    )

    return response(
        200,
        {
            "upload_url": upload_url,
            "object_key": object_key,
            "expires_in": URL_EXPIRES_SECONDS,
        },
    )
```

  Important: for buckets outside `us-east-1`, the returned presigned URL must use the regional S3 host, for example:

  ```text
  https://amzn-s3-rallybox-bucket-911.s3.eu-central-2.amazonaws.com/...
  ```

  If your Lambda returns:

  ```text
  https://amzn-s3-rallybox-bucket-911.s3.amazonaws.com/...
  ```

  then S3 can reject the upload with `IllegalLocationConstraintException` even though the signature request itself succeeded.

## Lambda Environment Variables

Set these in the Lambda configuration:

```text
S3_BUCKET=amzn-s3-rallybox-bucket-911
S3_REGION=eu-central-2
S3_PREFIX=tracks
SIGNER_API_KEY=YOUR_SIGNER_SECRET
URL_EXPIRES_SECONDS=300
```

If you want the path to become `tracks/rallyboxid/filename`, keep `S3_PREFIX=tracks`.

## Required Lambda IAM Permissions

Attach an execution role that allows presigning for objects under the desired prefix.

Example policy:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::amzn-s3-rallybox-bucket-911/tracks/*"
    }
  ]
}
```

### Exact AWS Console Steps For IAM Permission

This is the part that corresponds to the earlier step 15.

1. Open AWS Lambda.
2. Open your function `rallybox-gpx-signer`.
3. Click `Configuration`.
4. Click `Permissions`.
5. Under `Execution role`, click the role name link.
6. AWS opens the IAM role page in a new tab.
7. Click `Add permissions`.
8. Click `Create inline policy`.
9. Click the `JSON` tab.
10. Delete the default JSON.
11. Paste this exact policy:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::amzn-s3-rallybox-bucket-911/tracks/*"
    }
  ]
}
```

12. Click `Next`.
13. Policy name: `RallyBoxGpxSignerPutObject`.
14. Click `Create policy`.

What this does:

- it allows the Lambda to generate upload authorization for files only under:

```text
s3://amzn-s3-rallybox-bucket-911/tracks/*
```

- it does not grant access to the whole bucket
- it does not expose AWS keys to RallyBox

If your uploads must always go to:

```text
tracks/{rallybox_id}/{filename}
```

then this policy is the correct starting point.

## AWS Deployment Outline

1. Open AWS Lambda.
2. Create a new function.
3. Choose Python runtime.
4. Paste the code above into `lambda_function.py`.
5. Add the environment variables listed above.
6. Attach the IAM policy allowing `s3:PutObject` on `arn:aws:s3:::amzn-s3-rallybox-bucket-911/tracks/*`.
7. Enable a Function URL, or place the Lambda behind API Gateway.
8. If using a Function URL, note the final HTTPS URL.

### Exact AWS Console Steps For Function URL

This is the part that corresponds to step 5 in the short checklist.

1. Open AWS Lambda.
2. Open your function `rallybox-gpx-signer`.
3. Click `Configuration`.
4. Click `Function URL`.
5. Click `Create function URL`.
6. For `Auth type`, choose `NONE`.
7. Leave `Invoke mode` at the default unless AWS shows a different recommended setting.
8. Leave CORS disabled.
9. Click `Save` or `Create`.
10. AWS now shows a full HTTPS URL.

It will look roughly like this:

```text
https://abcde12345.lambda-url.eu-central-1.on.aws/
```

That full HTTPS address is your signer URL base.

### What To Do With That URL

Copy that URL and place it into RallyBox config like this:

```text
CONFIG_RALLYBOX_GPX_WEB_SIGNER_URL="https://abcde12345.lambda-url.eu-central-1.on.aws/?rallybox_id=rallyboxid&filename={filename}"
```

Replace:

- `https://abcde12345.lambda-url.eu-central-1.on.aws/` with your real Lambda URL
- `rallyboxid` with your real RallyBox folder name in S3

### Why `Auth type = NONE` Is Acceptable Here

You are already protecting the signer with your own header:

```text
x-api-key: YOUR_SIGNER_SECRET
```

So AWS Function URL authentication does not need to do the protection for you.
The Lambda code itself checks your secret before returning an upload URL.

## PC Test Before Using RallyBox

First test the signer endpoint:

```bash
curl \
  -H "x-api-key: YOUR_SIGNER_SECRET" \
  "https://YOUR_LAMBDA_URL/gpx-sign?rallybox_id=rallyboxid&filename=21042026-124327.gnss.gpx"
```

The response should contain `upload_url`.

Then test the returned presigned URL directly:

```bash
curl -X PUT \
  -H "Content-Type: application/gpx+xml" \
  --data-binary "@test.gpx" \
  "PASTE_UPLOAD_URL_HERE"
```

If that works, RallyBox should also be able to upload.

## Troubleshooting

If RallyBox logs:

```text
Signer request failed: http=502 body=Internal Server Error
```

then the Lambda Function URL is reachable, but the Lambda code itself is throwing an exception before it returns JSON.

Use this exact check:

1. Open AWS Lambda.
2. Open your signer function.
3. Click `Monitor`.
4. Click `View CloudWatch logs`.
5. Open the newest log stream.
6. Read the Python traceback from the failing request.

Then run a Lambda test event directly in the AWS console with this shape:

```json
{
  "headers": {
    "x-api-key": "YOUR_SIGNER_SECRET"
  },
  "queryStringParameters": {
    "rallybox_id": "rallyboxid",
    "filename": "21042026-143649.gnss.gpx"
  }
}
```

Expected result:

- Lambda returns status `200`
- response JSON contains `upload_url`
- `upload_url` host contains the regional endpoint, for example:

```text
amzn-s3-rallybox-bucket-911.s3.eu-central-2.amazonaws.com
```

If it still returns the global host:

```text
amzn-s3-rallybox-bucket-911.s3.amazonaws.com
```

the signer is still generating the wrong endpoint for your bucket region.

## Important Notes

- `SIGNER_API_KEY` is a secret you create. AWS does not create it for you.
- The Lambda URL is the HTTPS endpoint AWS gives you when you enable Function URL or API Gateway.
- RallyBox should not store raw AWS access keys for this flow.
- If your previous AWS keys were exposed in `sdkconfig`, rotate them.
